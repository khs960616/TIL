## 라우터에서 발생할 수 있는 packet delay의 요소들
```
1. Nodal Processing : 패킷이 최초에 들어오면 해당 패킷이 손상되었는지(bit error 확인), 목적지는 어디인지를 확인해서 라우터의 어떤 네트워크 인터페이스로 나가는가 결정해야한다. (라우터의 성능을 높이는 것으로 개선 가능하다.)

2. queueing : 나가고자하는 네트워크 인터페이스에서는 네트워크 혼잡등을 고려해서 데이터를 전송하므로, 잠시 queue에 해당 패킷을 보관한다.

(라우터에 outgoing link로 패킷을 내보내는 속도보다 들아오는 속도가 더 빠른 경우를 대비) >> 건드리기 어려운 영역임. 일반적으로 대다수의 패킷 유실이 해당 부분에서 일어난다.

3. Transmission delay : 패킷(데이터)은 결국 연속된 bit의 집합이다. 해당 패킷의 첫번째 bit~ 마지막 bit까지 링크를 통해 빠져나가는데 걸리는 시간 (패킷의 크기를 link의 bandwidth로 나눈 것으로 쉽게 정의된다)

(bandwidth가 높은 케이블로 link를 변경하면 줄어든다.)

4. Propagate delay : 패킷의 마지막 bit가 나가서 다음 hop까지 도달하는데 걸리는 시간 (광속으로 움직이기 때문에 줄이는게 불가능하다.)
```
